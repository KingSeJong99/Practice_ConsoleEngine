Game 클래스
- Game 클래스는 아마도 Engine 클래스의 자식으로 Level을 컨트롤 하는 것으로 보인다
- 그렇기에 내부 변수로 레벨을 관리할 배열이라던지, 관리에 사용할 열거형 클래스 등이 있다
- Level을 관리하는 것이 목적이기 때문에 Game 클래스는 인스턴스가 하나로 충분하여 정적 변수로 초기화한다(다른곳에서 생성안함!)

레벨을 관리하기 위한 열거형 enum class
- 우리가 만들 프로젝트는 현재 소코반 맵 한 개와 'ESC' 키를 눌렀을 때 나오는 메뉴 화면 총 두 개이다
- 이 레벨을 관리할 배열이 vector인 것에 대해 의아하긴 하지만 아마도 일반적인 프로젝트에서는 동적으로 맵을 늘려야 하는 상황이 발생할 수 있기 때문에 해당 프로젝트와 관계없이 vector로 선언된 것으로 보인다

Game 클래스의 생성자/소멸자
- SokobanLevel와 MenuLevel을 벡터 배열에 대입한다
- Game 클래스가 어떤 Level을 불러올지 설정한다
- Game 클래스가 참조했던 값들을 nullptr로 지정하고, clear를 통해 제거한다

ToggleMenu 함수
- Level이 늘어나면 의미가 없을 것 같지만, 2개 밖에 없는 특수한 상황에서 교육자는 1 - x 기법을 통해 토글 기능을 수행하는 방법을 소개하였다

궁금한점
- 클래스 내에서 해당 클래스의 주소를 갖는 포인터 변수를 private 멤버 변수로 선언한 다음 모듈에서 nullptr로 초기화 하는 부분이 있는데, 이것은 포인터 변수가 '무엇을 참조하는지'에 대한 정보만 갖고 있으면 되기 때문에 클래스가 생성되지 않은 상태에서도 멤버 변수인 포인터를 초기화할 수 있는 것일까? 그리고 그것이 생성자에서 this를 통해 대입이 이루어지는 것일까? 이러한 행위가 갖는 의미는 무엇일까?
