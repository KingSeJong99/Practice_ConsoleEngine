SokobanLevel 클래스
- SokobanLevel 클래스는 Level 클래스와, ICanPlayerMove를 상속받는다
- SokobanLevel은 한 게임의 무대 역할을 하며 그에 관련된 역할을 수행한다
- 그리기, 맵을 불러오기, 움직임(제어), 클리어 확인하기의 역할을 가진 함수를 갖고 있다

Draw 함수
- 기본적으로 Level 클래스의 Draw 함수를 상속받아 오버라이딩 한다
- 이때 SokobanLevel은 게임 내에서 게임 클리어를 독자적으로 출력할 수 있다

LoadMap 함수
- 우리 프로젝트는 Console 기반 게임 엔진으로, 모든 렌더링이 기호와 문자를 통해 이루어진다
- 이를 화면에 출력하기 위해서 배열에 정보를 저장하여 전달을 해야하는데 코드에 직접 작성하게 될 경우 코드가 간결해지지 않아 일반적으로 텍스트 파일과 같이 따로 저장을 한 다음 해당 포맷을 파싱을 하게 된다
- 바로 sprintf 함수를 이용해, 해당 파일의 주소를 변수를 통해 저장할 수 있다
- 이 저장한 주소값을 fopen 함수에게 전달하면 file 포인터에게 전달 할 수 있다
- fseek 함수를 이용하면 파일의 포지션 포인터를 파일의 끝으로 이동시킬 수 있다. 이를 통해 파일에서 데이터를 읽어올 버퍼를 동적으로 할당할 수 있다
- 마지막으로 fread 함수를 이용하면 특정 변수에 문자열 형태로 값이 저장된다
- 이것을 파싱(분석)하여 화면에 출력을 하면 된다
- 간단하게 맵이 구성된 경우 switch-case문을 통해 그것에 맞는 Actor들을 호출하면 그리기가 끝난다
- 즉, 벽이나 플레이어, 또는 다른 각종 오브젝트들이 기재된 파일을 불러와서 생성을 하는 것이다. 벽이 만약 200개라고 하면 벽 개체는 해당 레벨에 200개가 생성되는 것이다!
- 그리기가 끝났다면 디스크에서 메모리로 옮기는데 사용되었던 버퍼를 해제(delete)한 후 파일을 닫으면 된다

CanMove 함수
- Level 클래스는 Level 위에서 활동하는 객체들에게 규칙을 부여하는 것과 같은 역할을 한다
- 따라서 Level은 플레이어가 벽에 닿았을 때, 박스에 닿았을 때에 대한 처리를 하며 플레이어에게 피드백 해야 한다
- 불필요한 for문 검색이 너무 많아 리팩토링이 필요해보인다

CheckGameClear 함수
- CheckGameClear 함수는 소코반의 목적에 맞게 박스가 지정 위치에 도달했는지 검증을 해야한다
- 함수는 액터들을 찾아다니며 박스와 타겟을 찾아 배열에 추가한 후 각각 좌표가 맞는지를 판단하고 있다
- 함수가 호출될 때마다 currentScore을 0으로 초기화하는데 이것은 currentScore가 무한으로 뻥튀기 되는 것을 막기 위한것일까?
