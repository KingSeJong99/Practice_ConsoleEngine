Engine 클래스
- Engine 클래스가 1차와 2차에 갖고 있던 기능의 대부분을 Input의 클래스에게 옮기며 Engine의 코드가 다이어트 되었다.
- Engine 클래스는 그런 Input 클래스를 전방 선언하며 이용하는 것이 Engine 3차에서의 차이이다.

Engine은 Input 클래스의 friend.
- Input 클래스가 Engine 클래스를 friend 지정하였기 때문에 접근이 가능하다. 

Engine 클래스는 하나의 객체로 충분하다
- Engine은 Level 과 Actor를 아우르는 총관리자 역할로서 그 존재는 하나로 충분해 디자인 패턴 중 하나인 싱글턴을 적용한다
- 따라서 전역변수의 역할을 하는 static Engine* instance 변수(속성)을 추가한다. 그리고 이를 접근하기 위한 함수도 추가한다
- 생성자를 통해 스태틱 변수의 주소값에 인스턴스의 주소값을 대입하고 이번에 추가한 Input 클래스를 입력 관리자로서 생성한다

Engine 클래스가 퇴장하면..
- Engine 클래스가 수명을 다하고 나면 기존의 mainLevel을 지우고 nullptr을 지정하던 것에 이어 입력 관리자도 그 자격을 박탈하는 것이 중요하다. 따라서 Input 클래스도 delete를 통해 존재를 지우고 nullptr을 지정하도록 한다

Input이 있으니 이제 Run 함수를 수정해보자
- 게임루프의 핵심인 입력/틱/그리기 중 하나인 입력의 역할을 하는 Input이 생겼으므로 Input 클래스가 가진 메소드를 처리하도록 한다. 이때 피드백을 해야하기 때문에 Input 클래스는 현재의 상태값을 저장해야 한다
