Engine 클래스
- 여태까지 구현된 Engine은 고정 프레임을 사용하기 위해 프레임 속도를 코드 안에서 정의하였다
- 하지만 파일 입출력을 통해 고정 프레임의 값이 저장된 텍스트 파일로 코드 밖에서 프레임을 조절할 수 있게 되었다

Engine 클래스의 변경된 생성자
- 프레임 속도가 기입된 파일을 저장하는 LoadSetting() 함수를 실행한다
- 콘솔창에서의 문제는 사용자 입력을 대기 받기 위해 커서가 수시로 깜빡인다는 점이다 이를 막기 위해 windows API를 이용한다

파일을 받아들이며 달라진 Run 함수
- Engine 클래스는 이제 파일의 값에 따라 프레임 속도가 조절되기 때문에 유효 값을 체크하고 파일에 저장된 속도로 고정프레임을 진행한다
- 'q' 입력시 OFF로 돌려놓았던 커서를 다시 깜빡이도록 바꿔준다

LoadSetting 함수(파싱)
- 파일을 읽어들일 파일 포인터를 만들어 지정된 위치에 있는 텍스트 파일을 읽기 모드로 불러온다
- fread 함수를 통해 파일의 값들을 메모리에 적재한다
- 저장된 값 중 문자열 포맷을 활용해서 데이터를 추출한 후 파일을 닫는다
(ifstream과 string, fscanf으로 파싱하는 방법도 있다)

CONSOLE_CURSOR_INFO? bVisible? 등의 정체..
- bool 타입으로 false면 커서가 사라지고, true면 커서가 나타난다
- dwSize는 커서의 두께를 의미한다
- SetConsoleCursorInfo 호출시 GetConsoleCursoInfo로 현재 정보를 받아온 후 bVisible만 바꿔야한다
