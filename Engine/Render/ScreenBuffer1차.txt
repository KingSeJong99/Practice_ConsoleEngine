ScreenBuffer 클래스
- console 환경에서 자연스럽고 효과적으로 매 틱당 문자들을 출력하기 위해서는 '더블 버퍼링'이라는 기법을 적용해야 한다
- 더블 버퍼링은 두 개의 버퍼를 준비해 한 개의 버퍼가 한 틱에 화면에 출력되는 동안, 뒤에서 다음 틱의 장면을 draw하여 다음에 곧바로 제출할 수 있도록 하는 효율적인 협력 기법이라고 할 수 있다
- winnt 헤더파일에 존재하는 HANDLE 매크로를 사용한다

생성자
- ScreenBuffer 클래스는 아마도, 커널에게 권한을 얻어 콘솔에 대한 제어권을 일부 얻는 것이다 이에 따라 Win 개발 환경에서 다음과 같은 함수를 사용해야 한다

CreateConsoleScreenBuffer
- 함수를 이용해 Console Output을 버퍼에 저장한다
- 함수 호출이 성공하면 새 콘솔 화면 버퍼에 대한 핸들을 제공한다 콘솔에는 여러 화면 버퍼가 있지만 활성 화면 버퍼는 하나만 있을 수 있기 때문에 새 화면 버퍼를 활성 화면 버퍼로 만들기 위해서는 SetConsoleActiveScreenBuffer 함수를 사용해야 한다
- 버퍼 생성을 실패할 수 있기 때문에 해당 반환값을 통해 예외를 처리할 수 있어야 한다
- 콘솔의 크기는 short 크기 변수 4개로 이루어진 구조체를 사용해 좌상단과 우하단의 범위를 지정해야한다(왜 unsigned 형이 아닌걸까?)

소멸자
- 권한을 얻었던 버퍼를 반납해야 한다, CloseHandle 기능을 사용해야한다

Clear 함수
- Clear 함수는 FileConsoleOutputCharacterA 함수를 통해 지우기 기능을 한다

FileConsoleOutputCharacterA
- 작동할 콘솔, 입력할 문자, 문자의 수, 출발 좌표와 사용한 글자(왜지?)를 매개변수로 입력받아 지정한 영역에 화이트 스페이스를 넣어 마치 다 지워낸 것 처럼 만든다

Draw 함수
- Draw 함수는 WriteConsoleOutputA 함수를 통해 버퍼에 받았던 글자 배열들을 찍어내는 역할을 한다
