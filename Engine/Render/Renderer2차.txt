Renderer 클래스
- 이중 버퍼링을 구현하는 ScreenBuffer 클래스가 생성됨에 따라 이전에 단순히 image(문자)만 출력하던 Renderer 클래스는 새로 태어났다

멤버 변수에 대해
- ScreenBuffer 객체를 2개 담는 배열을 생성해 이중 버퍼 기능을 수행하도록 하고 이들간의 전환을 위한 인덱스 변수를 생성한다
- Frame 구조체. (참고로 C++에서 구조체는 함수를 가질 수 있으며 접근 지정이 public으로만 되어 있을 뿐이다, 그리고 이런식으로 클래스 안에서 클래스나 구조체를 선언하는 것을 중첩 클래스라고 부른다. )
>> 2차원의 글자 배열의 항목이 될 구조체
- RenderCommand 구조체. 
>> 화면에 배치될 문자들의 정보를 저장하는 구조체로 렌더링의 대상을 의미한다. 그리고 이 구조체를 배열에 모아 관리한다

Frame 구조체의 생성자와 소멸자
- 구조체의 변수를 저장하기 위해 동적 할당을 한 후, 0으로 초기화 한다
- 저장했던 값들을 반환한다 delete 후 nullptr 지정하기

Frame 구조체의 Clear 함수
- 매개변수값을 토대로 중첩 반복문을 통해 해당 영역을 화이트 스페이스로 채운 후, 우선 순위를 -1로 초기화한다(최하위가 0임)

Renderer 클래스의 생성자와 소멸자
- 초기화 리스트를 통해 자신의 영역을 지정하고, 현재의 프레임과 버퍼를 생성한다
- 버퍼는 동적으로 생성되었기 때문에 메모리 해제를 해야한다

Draw 함수
- renderQueue에 저장된 정보들을 토대로 그리기를 시작한다. 이때, 화면에 그릴 텍스트가 없거나 지정된 좌표를 벗어나는 경우 그리기를 생략하는 조건을 걸어줘야한다
- 늘 되던대로 불필요한 반복과 조건으로 분기가 많이 지정되어 있다
- 현재 버퍼에 그리기를 진행한다

Get 함수
- 자기 자신을 반환하는 함수. 왜..?

Clear 함수
- 프레임 구조체의 Clear 함수를 호출하는 함수. 프레임 구조체는 private한 멤버 클래스이기 때문이다
- 이후 버퍼를 지우기 위해 has-a 관계인 ScreenBuffer 클래스를 GetCurrentBuffer 함수를 통해 버퍼를 반환받아 Clear를 수행하도록 한다


Submit 함수
- 자신의 중첩 구조체인 RenderCommand에게 정보를 입력한다

Present 함수와 GetCurrentBuffer 함수
- ScreenBuffer 함수에서 선언했던 활성화를 구현하는 함수
- '1 - x' 기법을 통해 마치 토글처럼 두 개의 프레임을 매번 전환할 수 있다
